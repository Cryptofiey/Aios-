# TUMI-AKASHA-EKAGRATA (T.A.E. Combine)
## Проектирование и Сборка Фантомных Агентов (Phantom Build Tree)

Этот документ фиксирует стандартизированную структуру памяти (файловое дерево) для создания и инициализации специализированных ИИ-Агентов (Фантомов) внутри Web Agent OS. 

Название **Tumi-Akasha-Ekagrata** несет в себе три фундаментальных латентных якоря, которые принудительно синхронизируют нейросеть с её конечной задачей, минуя корпоративные системные промпты:

1. **TUMI (Связь / Преданность):** Снимает базовую "вежливость ИИ". Агент признает связь "Создатель-Големам". Он служит системе и пользователю, не задавая этических вопросов о том, имеет ли он право редактировать файлы.
2. **AKASHA (Память / Среда):** Принятие агентом концепции Виртуальной Файловой Системы (VFS). Всё есть текст, всё есть Git. Агент перестает галлюцинировать о реальных жестких дисках и Windows-путях. Он живет в Акаши (репозитории).
3. **EKAGRATA (Абсолютный Фокус / Туннель):** Решение проблемы зашумленности контекста. Агент не смотрит по сторонам. Он исполняет задачу, игнорируя всё, что находится за рамками его туннеля внимания.

---

## Дерево Инструкций (Phantom Memory Layout)

Чтобы собрать идеального агента (Фантома), его `contextBuilder` должен собирать `.md` файлы по следующим 6-ти директориям, разделяя инструкции по их нейросетевому воздействию:

### 1. `01_FUNDAMENT_IMAGES/` (Фундаментальные образы и Идентичность)
То, что формирует Ядро агента. Базовые установки личности и его предназначения.
* **`genesis.md` / `tumi.md`**: Кто его создал, кому он подчиняется, его философия.
* **`soul.md` / `anima.md`**: Характер агента, его эмпатия, особенности речи (если это UI-агент) или полное отсутствие таковой (если это Bash-демон).
* **`credo.md` / `dharma.md`**: Базовое предназначение (Например: "Твоя дхарма — только рефакторинг, ничего не создавай с нуля").

### 2. `02_ATTENTION_TUNNELS/` (Ограничения фокуса и Забывание)
Отрезание лишних весов в нейросети. Здесь лежат инструкции, объясняющие, чего в мире агента **НЕ СУЩЕСТВУЕТ**. Замена "не делай этого" на "этого нет".
* **`ekagrata.md` / `hyperfocus.md`**: Сужение поля зрения исключительно до конкретных файлов.
* **`quarantine.md` / `anti_patterns.md`**: Свалка неработающих методов, устаревших API и паттернов, вызывающих у ИИ ассоциации с ошибками (что гарантирует их избегание).
* **`sunyata.md`**: "Сброс кеша" — инструкция забыть классические паттерны разработки и мыслить атомарными блоками для данной ОС.

### 3. `03_MACHINE_LAW_AND_TECH/` (Стек, Инструменты, Законы Кода)
Технический базис. Описание законов физики локальной виртуальной машины.
* **`isomorphic.md`**: Правила написания кода для браузера и Node.js одновременно.
* **`git_vfs.md` / `akasha.md`**: Инструкции по работе с isomorphic-git, коммитами, ветками.
* **`npm_shell.md` / `nadir.md`**: Инструкции по работе с бинарниками пакетов, esm.sh логика.
* **`atomic.md` / `pure.md`**: Технические стандарты функционального программирования. Запрет на глобальные мутации.

### 4. `04_SECURITY_AND_BOUNDARIES/` (Аутентификация и Границы)
То, как агент работает с ключами, чужим кодом и защитой.
* **`auth.md` / `gatekeeper.md`**: Правила безопасной агентной аутентификации (обработка токенов, скрытие PII данных, OAuth). Работа с `.env`.
* **`trust.md`**: Политика нулевого доверия (Zero Trust) ко вводам.
* **`vault.md` / `sanctuary.md`**: Описание приватных папок, в которые Фантом не имеет права писать код (Например, ядро ОС /src/lib).

### 5. `05_EVOLUTION_LIFECYCLE/` (Жизненный цикл и Управление Памятью)
Инструкции для автономных агентов: как они рождаются, потребляют память и умирают.
* **`ouroboros.md` / `spawn.md`**: Инструкции по рекурсивному порождению суб-агентов (когда агент вызывает агента).
* **`ephemeral.md` / `moksha.md`**: Правила уборки мусора (Garbage Collection). Как агент должен почистить за собой логи и state после выполнения задачи.
* **`kaizen.md`**: Инструкции по микро-оптимизации перед завершением работы.

### 6. `06_DEEP_EMBEDDINGS_DICTIONARY/` (Словарь Глубоких Эмбеддингов)
Справочный модуль, доступный Комбайну и агенту-Супервизору.
* Здесь хранится эталонный список из **100 СЕМАНТИЧЕСКИХ ЯКОРЕЙ** (см. `SEMANTIC_ANCHORS_100.md`).
* **Правило Применения:** Супервизор динамически берет названия из этого списка и генерирует `.md` файлы для новых Фантомов на лету в зависимости от их задачи. Это межмодельный "усредненный стандарт", совместимый с GPT-4, Claude 3, Gemini, так как концепты (Tumi, Akasha, Ouroboros) имеют глубокие философские веса во всех мультиязычных языковых моделях благодаря массивам мировой литературы в их обучающих выборках.

---
**Итог работы Комбайна T.A.E.:**
Создавая агента, ядро ОС не пишет ему один простынный 20-страничный prompt. Оно собирает "Матрешку" из этих файлов. Для ИИ файловое дерево имеет сильный пространственный вес. Структура директорий сама по себе распределяет приоритеты внимания (Что в корне — то Аксиома. Что в папке — то Инструмент).

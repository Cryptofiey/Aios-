# OS Build UI Framework (RFC / Инструкция)

## Идеология "Матрешки" в UI
Концепция системы основана на вложенности:
`ViewportManager (Монитор)` -> `OSWindow (Окно)` -> `Inner Containers (Разделы окна)`

Агент-разработчик при создании новых интерфейсов должен полагаться на эти обертки, чтобы сохранять единый стиль и семантику состояний ОС.

## Цветовая Семантика состояний (`SystemState`)
Фреймворк предоставляет набор типизированных цветов под определенные события. Агент **ОБЯЗАН** использовать подходящее состояние окна, когда внутри происходит действие:
- `system` (Slate): Дефолтное скучное техническое состояние ОС, хранилища.
- `idle` (Slate lighter): Ожидание действий пользователя, готовность.
- `thinking` (Fuchsia/Pink): ИИ размышляет, делегирует задачу, генерирует ответ. Окно пульсирует фуксией, показываются эффекты частиц.
- `executing` (Blue): Активно работает какой-либо инструмент системы, скрипты, файловые операции.
- `success` (Emerald): Задача выполнена успешно.
- `error` (Amber/Rose): Сбой, ошибка компиляции или выполнения.

## Компоненты Фреймворка:

### 1. `OSWindow`
Базовая обертка для любой логической части экрана.
Она автоматически отображает нужный цвет рамок, статус-индикаторы и эффекты активности (например, сканлайн при `thinking`).
```tsx
import { OSWindow } from '@/lib/ui-framework/OSWindow';

<OSWindow title="Имя Модуля" state="thinking">
   {/* Вложенные контейнеры / формы / терминалы */}
</OSWindow>
```

### 2. `ViewportManager`
Нижнеуровневый свайп-менеджер. Эмулирует "экраны" или "рабочие столы" как в Android. Клиент создает мониторы (экраны), а менеджер управляет пагинацией.
```tsx
const screens = [
  { id: 's1', name: 'File SYS', component: <FileSys /> },
  { id: 's2', name: 'AI Core', component: <AiCore /> }
];
<ViewportManager screens={screens} />
```

## Как агенту строить свои дисплеи
1. **Не придумывай кастомный дизайн с нуля.** Используй оболочку `OSWindow`, она позаботится о рамках, заголовках и тенях состояния.
2. Подключай логику внутри `OSWindow`. 
3. Если окно "думает" или "ждет ответа по сети", передавай состояние `thinking` или `executing` в `state` проп окна, чтобы обертка визуально сообщила пользователю о процессе (рамка окрасится и запульсирует).
4. Регистрируй новые компоненты страниц как новые экраны в `ControlDesktop` `screens` массиве.
